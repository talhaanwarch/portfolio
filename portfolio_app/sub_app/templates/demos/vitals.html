Heart Rate detection via mobile camera

This model is being trained on a small public available dataset, so this is not 100% accurate. Though this model has high potentital and expected error is rate is around 10 bpm.

In order to get your heart rate upload video of 10-15 seconds.

We are not using any face detection algorithm as initial screening layer. We assume that every one using this demo is capturing the face. If some thing else is captured, it will still show results, but they will be incorrect.

The light should be sufficient and the head motion should be minimum to get accurate results.

Notes

When first time video is captured or uploaded, the server spins up, which will take some time. The server duration is 15 minutes and it will be down after 15 minutes and a new instance is started. This is one of the reason for slow response but its still good enough for demo purpose.
